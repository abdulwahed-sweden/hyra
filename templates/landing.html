Key Decisions

  • select_for_update() inside transaction.atomic() — prevents concurrent queue processing
  • Elasticsearch with silent Postgres fallback — search always returns results, response includes engine field
  • Denormalized applicant snapshots — queue entries freeze applicant data at application time
  • Reproducible lottery — random.Random(listing.pk) ensures auditability
  • Explicit eligibility pipeline — six rules checked in priority order, each independently testable
  • Webhook events with retry — landlord systems notified on queue.processed and tenant.selected; exponential backoff 5min→7days; HMAC-SHA256 signed
  • Redis caching — stats endpoint cached 60s to prevent DB pressure from high-traffic reads
